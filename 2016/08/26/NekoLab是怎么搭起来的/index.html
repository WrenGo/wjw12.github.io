NekoLab是怎么搭起来的

发表于 2016-08-26   |  

一些朋友问过我这个主页是怎么搭起来的。简单地说，是用Hexo生成静态页面，挂在Github上，视频和图片等多媒体文件存在阿里云的OSS云储存，通过外链使用。下面分别来说一说，也是给自己做个备忘。

Hexo

Hexo基本的安装已经在NekoLab新的起点这里写到了，不再赘述。

配置文件

配置Hexo经常要跟两个config.yml配置文件打交道，一个在根目录下，配置站点总体说明、插件等等；还有一个是主题的配置文件，位于themes/(主题名)/目录下。

比如说我用的NexT主题，这里主要配置了导航栏指向的页面和图标：

# When running hexo in a subdirectory (e.g. domain.tld/blog)
# Remove leading slashes ( "/archives" -> "archives" )
menu:
  home: /
  #categories: /categories
  about: /about
  archives: /archives
  tags: /tags
  friends: /friends
  video: /video
  rss: /atom.xml
  #commonweal: /404.html

# Enable/Disable menu icons.
# Icon Mapping:
#   Map a menu item to a specific FontAwesome icon name.
#   Key is the name of menu item and value is the name of FontAwsome icon.
#   When an question mask icon presenting up means that the item has no mapping icon.
menu_icons:
  enable: true
  # Icon Mapping.
  home: home
  about: user
  categories: th
  tags: tags
  archives: archive
  commonweal: heartbeat
  friends: group
  video: tv
  rss: rss

还需要配置导航栏的文字，在(主题目录)/languages/下有不同语言的配置，我的站点是简体中文，这里就应该编辑zh-Hans.yml：

menu:
  home: 首页
  archives: 归档
  categories: 分类
  tags: 标签
  about: 关于
  search: 搜索
  commonweal: 公益404
  friends: 友链
  video: 视频(测试版)
  rss: 订阅

新增页面

执行命令hexo new page就会在站点的source文件夹里生成一个新文件夹，内有index.md，编辑这个和编辑博文的markdown源文件方法一样。

要将这个页面增加到导航栏，需要修改上面主题的config.yml和语言相关的yml配置，在menu模块增加新页面的名字和链接。

RSS订阅

使用了hexo-generator-feed插件，会在根目录下生成一个atom.xml文件，就可以被RSS阅读器自动识别了。

Latex数学公式输入

使用了hexo-math插件，以下是测试：

Inline

Schrodinger’s equation is $i\hbar \frac{\partial \Psi}{\partial t}=-\frac{\hbar^2}{2m}\nabla^2\Psi + U\Psi$

Block

$$\frac{\partial u}{\partial t}
= h^2 \left( \frac{\partial^2 u}{\partial x^2} +
\frac{\partial^2 u}{\partial y^2} +
\frac{\partial^2 u}{\partial z^2}\right)$$

Inline with Tag

This equation $\cos 2\theta = \cos^2 \theta - \sin^2 \theta = 2 \cos^2 \theta - 1$ is inline.

Multiple Line with Tag

$$\begin{eqnarray} \nabla\cdot\vec{E} &=& \frac{\rho}{\epsilon_0} \\ \nabla\cdot\vec{B} &=& 0 \\ \nabla\times\vec{E} &=& -\frac{\partial B}{\partial t} \\ \nabla\times\vec{B} &=& \mu_0\left(\vec{J}+\epsilon_0\frac{\partial E}{\partial t} \right) \end{eqnarray}$$

阿里云OSS

图片、音频、视频等文件太大，不可能存在Github上，于是我用了阿里云的云储存服务OSS，可以生成外链插入自己网页中。

计费规则主要包含空间和流量费用。空间是包月的，个人网站用最小的那种几十G，半年才10块钱。向云空间上传文件，以及每次打开网页加载图片会产生流量，按量计费，浏览量不多时也很便宜。
